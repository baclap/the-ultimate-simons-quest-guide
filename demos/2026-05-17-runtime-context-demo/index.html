Before And After

Dora proves layout context and palette context can diverge

The layout candidate is still 2:8:2, but the live palette selector uses $30=02, $50=00, and $51=83. The resolver now derives that from ROM tables.

Resolver Rule

The alias comes from a nested special screen record

The rule is table-driven: scan special screen records, then alias only when one starts inside an earlier same-marker special stream in the same object set.

Candidate Dora Woods Part 2 2:$A1AB = FE 0D ...
Containing stream Veros Woods Part 2 2:$A1A3 = FE 06 ...
Raw runtime submap $83 submap 03 + high-bit flag

Guardrail

Special records are not all aliases

Dabi and Aljiba also use special screen records, but they do not meet the same-marker containment rule. Their palette contexts remain direct.
