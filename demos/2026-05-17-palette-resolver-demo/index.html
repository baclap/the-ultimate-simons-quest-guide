Milestone

Palette selection now comes from the ROM's runtime tables

The atlas no longer treats Dabi's Path as a generic woods screen. It follows the same transfer-id path the game uses to reach palette bytes.

Discovery

The selector is a two-step table path

The game does not choose outdoor palettes from the layout index directly. It reads a transfer id from bank 2, then resolves that id through the fixed-bank transfer table.

Runtime contextobjset / area / submap
Bank 2$F7C5 selector pointers
Transfer id$22, $23, $26
Fixed bank$C895 transfer table
PRG palette16 background color bytes

Before And After

Dabi's Path now uses the cave palette from ROM

The old atlas inherited the objset-2 woods fallback. The resolver reads transfer id $26, which points to 4:$A00A.

Evidence

Three fixtures now line up with the selector model

Jova Woods and Dabi's Path resolve directly from their cv2r context. Dora Woods Part 2 exposes a context-alias gap: the saved runtime selector context is 2:0:3 while the layout candidate is 2:8:2.

Next

The next gap is runtime context mapping

The palette mechanism is decoded, but the atlas still needs a general way to map every cv2r layout candidate to the runtime selector context the game used. Dora proves that those tuples can diverge.
