Milestone

A recipe resolver now renders the atlas variants

The renderer uses audited ROM-derived recipes to produce validated fixtures, projected day/night variants, and diagnostic output for unresolved families.

Validated Core

Known save-state probes now render from recipes

These atlas images are generated from ROM layout data, ROM CHR, and ROM palette transfer tables. The save states only validate the recipe selection.

Day And Night

The same resolver can project palette variants

Validated families can now emit day and night renders without adding per-location palette overrides.

Projected And Diagnostic

Unknown areas are rendered, but clearly labeled

Projected output is useful evidence, not a final claim. Diagnostic output marks families that still need representative save states.

Recipe Inventory

The manifest records confidence for every render

Each row keeps the runtime context, CHR banks, palette address, and validation source visible so the renderer avoids hidden per-area fixes.

Next Demo Goal

Validate the remaining families, then compose the world

The next states should target projected or diagnostic families where visual output still looks suspicious.
