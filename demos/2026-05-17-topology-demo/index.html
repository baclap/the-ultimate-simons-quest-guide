Milestone

The atlas now has exterior route topology

The project can read ROM transition triples and explain how rendered exterior candidates connect. This is adjacency data for map composition, not final world-coordinate placement yet.

Schematic Route

A ROM-derived path from Jova to Castlevania

The path is generated from resolved area-boundary edges. Some edges are traversed in reverse because the graph is treated as walkable adjacency for this schematic.

Rendered Route Inventory

The path is backed by ROM-rendered atlas segments

Each area card below contains the actual layout-space PNGs already emitted by the atlas pass. The order comes from topology, not category sorting.

Transition Evidence

Every route hop keeps its raw transition bytes

The graph stores the three-byte boundary tuple, decoded marker kind, source record address, and resolved target. This keeps the route explainable for future PNG or canvas consumers.

Known Gap

Mansion doors are connected landmarks, but their template is pending

The topology knows where mansion-door exteriors sit in the world. Their current render template is still visibly wrong, so the graph marks them without pretending that artwork is final.

Next Demo Goal

Turn topology into true composition coordinates

The next step is to promote this graph into a coordinate solver: align connected segment edges, validate representative windows, fix mansion templates, and then add day/night palette variants.
